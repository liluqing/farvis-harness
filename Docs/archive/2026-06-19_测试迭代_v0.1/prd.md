## 需求背景

本次迭代的核心目标是验证 Harness 文档管理体系的完整流程。通过一个简化的功能需求，测试从迭代创建、文档维护、开发协作到归档同步的全链路。

## 需求描述

### 功能 1: 视频生成 API 增强（farvis-video）

**用户故事**: 作为平台用户，我希望在生成视频时能够指定视频的分辨率选项。

**验收标准**:
- [ ] API 支持 `resolution` 参数（720p, 1080p）
- [ ] 默认分辨率为 1080p
- [ ] API 文档更新

### 功能 2: Credits 消耗记录（farvis-credits）

**用户故事**: 作为运营人员，我希望能够查询某个用户在指定时间范围内的 Credits 消耗明细。

**验收标准**:
- [ ] 新增查询 API: `GET /api/v1/credits/usage`
- [ ] 支持 `user_id`、`start_date`、`end_date` 参数
- [ ] 返回消耗明细列表（包含视频生成、语音克隆等类型）

## 影响范围

- **farvis-video**: 修改视频生成 API
- **farvis-credits**: 新增消耗查询 API

## 非目标

- 不实现前端界面
- 不做性能优化
- 不处理国际化
